
					Configure Janus



Janus provide two commands: "server" & "version". 

In order to configure Janus, there are three ways powered by Cobra and Viper (two libraries to create CLI applications) : 


====> To use a Configuration file

The first way is to use a configuration file. By default, the configuration directory is "/etc/janus". 
If there isn't configuration file, Janus will check in the current directory.

The name of the configuration file is "config.janus.json".


====> To use environment variable

Janus can retrieve environment variables beginning by "JANUS_".

====> To use Flags

The second way is to use the flags. Use "--help" or "-h" to access to Janus help. 

Here are the flags available:

Usage:
  janus server [flags]

Flags:
  -c, --config string              config file (default is /etc/janus/config.janus.json)
  -d, --consul_datacenter string   The datacenter of Consul node
  -t, --consul_token string        The token by default
  -a, --os_auth_url string         will use the 1.1 *compute api*
  -p, --os_password string         The password to authenticate
  -x, --os_prefix string           Prefix of the user
  -r, --os_region string           The region name
  -i, --os_tenant_id string        The ID of the tenant
  -n, --os_tenant_name string      The name of the tenant
  -u, --os_user_name string        The username to authenticate

===========

Viper configuration keys are case insensitive. Note that Viper uses the following precedence order -  each item takes 
precedence over the item below it: 1)explicit call to Set  2)flag  3)env  4)config  5)key/value store  6)default
